FAQ

Common API licensing questions

Keep the answers short, commercial, and confidence-building.
What does API licensing include?
API licensing covers access to the LinkCraftor API under a defined commercial model, including usage structure, permitted deployment scope, and the appropriate onboarding path for your type of integration.
Is commercial usage allowed?
Yes. The API is positioned for commercial use, but the exact scope depends on the license type and whether your use case fits standard access or requires expanded rights.
Can I embed LinkCraftor into my own product?
In many cases, yes. Standard commercial embedding may be possible under the right access tier, while more extensive productized redistribution may require OEM or custom licensing.
Do I need a custom license for white-label usage?
Usually yes. White-label, resale-oriented, or rebranded deployment models generally belong in a more advanced commercial agreement rather than a basic standard access plan.
Is usage billed separately from the license?
This page is meant to position the licensing structure. Detailed usage billing and monetization breakdowns can live on the wider API monetization pages.
Can I start monthly and move to annual later?
Yes, that is a reasonable growth path. Teams can begin with a simpler entry structure and move into annual or negotiated agreements as volume and operational needs increase.
Do you provide sandbox access?
Yes. Sandbox access should be part of the evaluation and onboarding pathway so buyers can test integration logic before a production rollout.
